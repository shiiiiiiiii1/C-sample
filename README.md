## 第十一回
**三項演算子**	
if文を一行でスマートに描く方法。
```
if(条件) {
	return A;
} else {
	return B;
}
この処理を、三項演算子を使って書くと
(条件) ? A : B;
これは、真の場合 A, 偽の場合 B を返す。
```

## 第十二回
**変数の有効範囲**  
ローカル変数とは、その関数内でしか使えない。  
他の関数からアクセスするためには、関数の外に変数を定義する。  
外に出した変数をグローバル変数という。

## 第十三回
**変数の有効期限**  
関数を何回も呼び出した時に、その値を保持していて欲しい時には、関数内の変数に` static int a = 0;`といった風に書いてあげると何度も実行した時にその値を保持してくれる。

## 第十四回

## 第十五回
**文字列**  
C言語は`string`という概念がない。文字列を扱う時は`char`型の配列で文字を一文字づつ送るようなイメージ。char型の文字列の最後には必ず**'\0'**が、入ってくる。だから、記述方法としては、
```
char s[] = {'a', 'b', 'c', '\0'};
char s[] = "abc";
char s[4] = "abc";
```
どの記述方法も同じ意味を示す。

## 第十六回
**ポインタとメモリ空間**  
メモリには値を格納するところがあってそこの一つずつにアドレスが付いている。int型はアドレス4つ分の領域を使う。  
ポインタはアドレスを格納するためのもの。変数を直接いじって扱うのではなく、アドレスを直接いじって値を変化させていくというイメージ。

## 第十七回
**ポインタとメモリ空間2** 
ポインタ変数を指定する方法は、`int *変数名`と定義する。 
ここで重要なのは、ポインタの宣言の仕方が`int *`であるから、あくまで変数名は、その後に記述してあるものとなる。 
ポインタ変数に、アドレスを格納する場合には、`&`という**アドレス演算子**というものをつけて格納する。アドレス演算子をつけると、その変数に入っている値のアドレスを取ってくることができるということ。 
また、ポインタから`printf`する場合には、第二引数に`*pa`を記述する。これは、ポインタを宣言したときの`*`とは意味が違い、間接演算子と呼ばれるものでポインタを通って間接的に値を呼び出している。ということである。

## 第二十一回
**関数の渡し方**  
値渡し：引数に値を直接書いて値を直接渡すこと。  
特徴として、それぞれの関数の領域内でそれぞれの値を保持している。  
参照渡し：アドレスを渡す方法。  
特徴として、参照して値を編集しているので参照先の変数を直接編集している。
